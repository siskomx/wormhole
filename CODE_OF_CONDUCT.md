# Code of Conduct

## Our Standards

Wormhole is a technical project for evidence-aware coding-agent workflows. Project spaces should stay focused, respectful, and useful to contributors.

Expected behavior includes:

- Use welcoming and professional language.
- Assume good intent while asking for evidence and clarification.
- Give actionable technical feedback without personal attacks.
- Respect different levels of experience and different working contexts.
- Keep issues, pull requests, discussions, and reviews on topic.

Unacceptable behavior includes:

- Harassment, threats, insults, or discriminatory language.
- Sexualized language or imagery in project spaces.
- Publishing another person's private information without permission.
- Repeated disruption of project discussions after a maintainer asks for a change in behavior.
- Trolling, bad-faith arguments, or personal attacks.

## Scope

This code of conduct applies in all project spaces, including GitHub issues, pull requests, discussions, code review, release notes, and any community channels managed by the project.

It also applies when someone represents the project in public spaces.

## Enforcement

Maintainers may remove comments, close issues or pull requests, restrict participation, or ban contributors whose behavior violates these standards.

Enforcement decisions should be proportional, documented when practical, and focused on protecting the project community.

## Reporting

Report unacceptable behavior privately to a maintainer through GitHub. If the report involves a security issue, follow the process in `SECURITY.md` instead of opening a public issue.

Maintainers will review reports as promptly as practical and will avoid sharing reporter details beyond what is needed to investigate and respond.
